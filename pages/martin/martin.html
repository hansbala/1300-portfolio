Blog Redesign

In this project I aimed to achieve the following goals:

  • Identify an unresponsive website
  • Come up with a new and modern mobile-first design for the website
  • Code up my design using HTML and CSS (perhaps using some frameworks)
  • Test how this interface holds up

I ended up choosing George RR Martin's Blog website which can be found here

Below you can find the sketches and the mockup that I came up with.

I ended up using BootStrap as a markup framework to allow for that mobile responsiveness, and in the end I think it all came together well.

Usability Issues

I identified a couple of usability issues with the blog portion which I outline below:

  • The user has to read all the articles in their entirety before going to the next article. There is no way to skim through the headlines of artciles and this proves to be really cumbersome. This occurs in both the mobile and desktop versions.
  • The desktop version of the website does not take advantage of the ample amount of space available. The website has a single columned layout showing only one blog post at a time. As a user who uses a monitor, the website really threw me of.
  • Some items of the website do not appear on the mobile version of the website (such as the Calendar, tags, and recent posts).
  • While skimming through the website, it is difficult to identify the end of one article and the beginning of the next.
  • Social media links are not placed in the usual locations and I had to Cmd+F to find them!
  • The color contrast created with the white text on the grey background is a bit too simplistic and leaves the user wanting more.
  • The layout of the elements of the website seems very 90s like.

Accessibility Issues

While not too many, the website could use some improvements with regards to accessibility. They are as follows:

  • Some of the blog post images do not have alt texts associated with it.
  • The checker reported that quite a lot of generic elements lack enough color contrast.
  • Some form input labels were missing which caused screen readers to malfunction when arriving at that point in the page.
  • The screen reader seems to struggle with the headers of an individual blog post. Adding aria labels to this would solve this.

New Website - Not a Blog

Here are some observations I had while checking for responsiveness and so forth:

  • The website properly scaled for mobile, desktop, and tablet screen sizes.
  • I also ran the new website through the WAIVE accessibility checker and did not encounter any errors. I only came across a few suggestings but that included things like increasing contrast, which was already enough
  • I also tried to test the responsive behavior of the website in landscape mode for the tablet and mobile. Unsurprisingly, the landscape mode behaves like a narrower iPad version and scales accordingly.
  • I also used Google Translate to translate it into my native speaking language and it translated it pretty well other than some links that were embedded in the navigation bar :(
